# Learnings — simplify

> **Per-skill, cross-project memory** — lessons for skill `simplify` usable across every project
>
> **What to store here:**
> - Extraction patterns that work well (e.g. composable boundary that generalizes)
> - Anti-patterns — what looked like dead code but wasn't (dynamic import, test fixture, string ref)
> - Heuristics for when NOT to extract (e.g. 3 copies all in different domains)
> - tsc / lint signals that reliably find dead code
>
> **What NOT to store here:**
> - Project-specific files simplified (one-shot — git log has it)
> - Domain-specific simplification for one project
>
> **When to read:** every Pre-flight of skill `simplify`
> **When to append:** after simplification completes if a generalizable pattern/anti-pattern was found
